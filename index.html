📱 Labs

Lab 1 · UI Design

Building foundational layouts with modern design principles and tools.

This lab introduced me to the fundamentals of user interface design. I learned how layout structure, color harmony, and typography work together to create intuitive user experiences. Seeing my first design come to life was both exciting and motivating.

Lab 2 · Interaction

Handling user input, button events, and dynamic state management.

I explored how applications respond to user actions through event handling and state. Debugging interaction flows taught me to think critically from the user's perspective. I realized that small details like button feedback significantly impact overall usability.

Lab 3 · Material Design

Implementing Material Design theming, components, and accessibility standards.

Material Design provided a structured framework for building visually consistent interfaces. I learned the importance of elevation, motion, and accessible color systems. This lab transformed how I approach design decisions in every project.

Lab 4 · Navigation

Multi-screen navigation architecture and safe data passing between destinations.

Setting up navigation between screens challenged me to organize application architecture logically. I learned to pass data safely between destinations and manage navigation graphs. This skill is essential for building any multi-feature application.

Lab 5 · Room Database

Local data persistence using Room, entities, DAOs, and coroutines.

Room made local storage surprisingly manageable once I understood entities, DAOs, and relationships. Using coroutines kept database operations smooth and non-blocking. I now feel confident building applications that function reliably even without internet connectivity.

🚀 Project

CommunityConnect

Problem Statement: Residents lack a centralized platform to report neighborhood issues and connect with local community services efficiently.

Building on Lab 4's navigation concepts, I developed a fully functional community reporting app. Integrating multiple screens with shared data required careful architecture planning and state management. This project taught me how even simple digital tools can strengthen neighborhood bonds and improve urban living for everyone.

UrbanPulse

Problem Statement: City dwellers need real-time information about local environmental conditions and public space availability to make healthier daily choices.

This project combined Room database from Lab 5 with sensor APIs and cloud synchronization for real-time data. Implementing asynchronous data streams pushed me to deepen my understanding of coroutines and API integration. The app demonstrates how mobile technology can empower citizens to make informed decisions about their urban environment.
